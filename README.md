# CPP11-ThreadPool

一、项目名称
C++11 高性能线程池（支持动态扩缩容与资源自动回收）

二、项目核心描述
基于 C++11 标准实现高可用线程池，支持任务异步提交、动态扩缩容、空闲线程自动清理，解决多线程任务调度的效率与资源浪费问题实现了线程安全的资源管理，避免内存泄漏和线程泄露；
核心采用 生产者-消费者 线程池模型，封装任务队列与线程状态管理，保证高并发场景下的线程安全与数据一致性；
适配 CPU 资源动态分配，在任务峰值时自动扩容（最多 12 线程），空闲时缩容至核心线程（2 线程），降低系统开销；
已通过 50+ 并发任务测试，无内存泄漏、死锁等问题，支持任意类型任务参数传入，可直接集成到服务端项目。


三、技术亮点
1. 并发编程核心实现
基于 std::thread std::mutex std::condition_variable 实现线程同步，解决多线程竞争、虚假唤醒等问题；
使用 std::atomic 原子变量管理线程数 / 任务数，避免锁开销，保证高并发下的数据一致性；
封装 WorkerStatus 结构体，通过「线程完成标志」精准管理线程生命周期，实现空闲线程自动回收。

2. 工程化设计与优化
任务队列采用「移动语义」（std::move）与智能指针（std::unique_ptr），避免内存泄漏，支持任意类型任务参数；
实现动态扩缩容策略：任务积压时批量扩容（每次 2 线程），线程空闲时自动缩容（保留核心线程），平衡效率与资源占用；
完善的异常处理机制（try-catch 捕获线程异常）与参数合法性校验，提升项目鲁棒性。

3. 技术选型与标准适配
基于 C++11 标准开发，不依赖第三方库，跨平台兼容（Windows/Linux），可直接集成到服务端项目；
使用 Visual Studio 2022 完成开发与调试，熟练掌握多线程调试技巧（线程 ID 跟踪、锁竞争排查）。

四、个人贡献
独立设计线程池架构，完成「任务封装、线程管理、扩缩容策略、资源回收」全流程开发；
解决核心问题：通过「线程完成标志 + 管理者线程定期清理」，实现已退出线程的资源精准回收，避免容器膨胀与内存泄漏；
优化锁竞争：将「全局锁」拆分为「队列锁 + 忙线程数锁」，减少多线程阻塞时间，提升并发效率；
编写完整测试用例，模拟 50+ 并发任务场景，验证线程池的稳定性与性能。
